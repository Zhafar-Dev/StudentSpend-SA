# QA Planning Notes

## System Goals
- The application should be simple and easy to use  
- It should perform well on low-end devices  
- It should provide clear expense categorisation  

## QA Approach
- Testing will be done after each feature is developed  
- Both functional and usability testing will be applied  
- Bugs will be logged and reported to developers  

## Focus Areas
- Input validation and error handling  
- User interface consistency  
- Performance and responsiveness  

## Observations (Planned)
- System should be tested for simplicity and ease of navigation  
- Special attention needed for budget calculation accuracy  
- Ensure categories are intuitive for users
